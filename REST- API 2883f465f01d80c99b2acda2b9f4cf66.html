  • API
    • An API, or Application Programming Interface, is essentially a set of rules and tools that allows different software applications to communicate and exchange information with each other
    • An API serves as an intermediary for applications, allowing them to communicate.
    • An API, or Application Programming Interface, allows developers to interact programmatically with applications instead of using traditional user interfaces like remote controls.
    • This concept can be compared to a remote control for a TV, bridging user actions with application responses.
    • Just as the waiter provides a clear way to order food and receive it back, an API defines a set of rules and protocols for how one application can request information or actions from another application, and how it will receive a response.
    • Popular API’s
      • YouTube
      • Google Maps
      • X
      • UBER

  • REST
    • Architectural style of the web
    • A standard/ set of guidelines by which we can structure & create API’s
    • REST API’s have identifiable properties

    • REST Properties
      • They make use of HTTP methods
        • GET → used to retrieve data from the server
        • POST → used to send data to the server
        • PUT → used to update data
        • DELETE → used to delete data
      • They make use of HTTP status codes

    • A RESTful API  organizes resources into a set of unique URIs, or Uniform Resource Identifiers. The URIs  differentiate different types of resources on a server.

    A REST implementation should be stateless. It  means the two parties don't need to store any information about each other, and every request  and response (cycle) is independent from all others. This leads to web applications that are easy to  scale and well behaved. 

  • HTTP Methods

    A way of telling the server what type of request we are making

    • GET
    • POST
    • PUT
    • DELETE

  • Endpoints
    • Endpoints are crucial in APIs, serving as specific URLs that developers can call to execute functions or retrieve data from the application.
    • They simplify interactions by providing a structured way to communicate with services.

    Body-parser is a middleware for Node.js applications, specifically those using Express.js, that parses the data in an incoming HTTP request's body. It converts the request body, which can be in formats like JSON, URL-encoded data, or raw text, into a JavaScript object that can be easily accessed on the req.body property.
